Automatic Deployment of DMS from Incidents

Incident Severity

Add a severity attribute to incidents. It can have a value of MINOR, NORMAL or MAJOR. The severity of an incident affects the distance to deploy DMS.

Severity Range Miles Branching
MINOR NEAR 1.5 No
NORMAL MIDDLE 5.0 Yes
MAJOR FAR 10.0 Yes

Find all DMS within the range of the incident. Each of these DMS will be deployed if the proper sign_messages can be found for selector, locator and advide.

Each sign_text message can be associated with one of the following incident categories: Locator, Selector or Advice.

Locator

A sign_text can be associated with an r_node, distance pair. Add a new MULTI tag to allow messages like X MILES AHEAD, for rural areas with no landmark nearby. Typically on a 3-line sign, these sign_text will be for line 2.

Distance Message Format
NEAR AHEAD
MIDDLE or FAR AT HWY 100
ALT_NEAR ON 394 EAST
ALT_MIDDLE or ALT_FAR ON 394 AT HWY 100

Incident Selector

An incident selector is a set of values which must match an incident to qualify for deployment. On a 3-line sign, a sign_text will normally have a selector for line 1.

  • Event type (CRASH, STALL, ROAD WORK, HAZARD)
  • Detail (Debris, Animal, Flash Flooding, etc.)
  • Lane type (mainline, exit ramp, etc.)
  • Clear flag

Advice

Non-blocking incidents?

  • IN x LANE
  • IN MEDIAN
  • ON x SHOULDER

For blocking incidents?

  • LANE CLOSED +variations
  • REDUCED TO x LANES
  • ROAD CLOSED
  • USE OTHER ROUTES

After blocking incident is cleared?

  • ALL LANES OPEN

Road work with all lanes blocked?

  • USE DETOUR

What criteria to use to select these?

  • REDUCE SPEED
  • USE CAUTION
  • PREPARE TO STOP
  • EXPECT DELAYS
  • MAJOR DELAY
  • LOCAL TRAFFIC ONLY
  • RADIO 88.5 FM
